基础

基础命令

数据类型​

Redis支持五种数据类型：string(字符串)、hash(哈希)、list(列表)、set(集合)、zset(sorted set：有序集合)

String（字符串）​

详细介绍

string 是 redis 最基本的类型，你可以理解成与 Memcached 一模一样的类型，一个 key 对应一个 value。

string 类型是二进制安全的。意思是 redis 的 string 可以包含任何数据。比如jpg图片或者序列化的对象。

string 类型是 Redis 最基本的数据类型，string 类型的值最大能存储 512MB。

命令

set key value [expiration EX seconds|PX milliseconds] [NX|XX]
get key

实例

127.0.0.1:6379> set name ovimRedis
OK
127.0.0.1:6379> get name
"ovimRedis"

在以上实例中我们使用了 Redis 的 set 和 get 命令。键为 name，对应的值为 ovimRedis

注意:一个键最大能存储512MB。

Hash（哈希）​

详细介绍

Redis hash 是一个键值(key=>value)对集合。

Redis hash 是一个 string 类型的 field 和 value 的映射表，hash 特别适合用于存储对象。

命令

hmset key field value [field value ...]
hget key field
hgetall key

实例

127.0.0.1:6379> hmset ovimHashKey name ovim desc engineer
OK
127.0.0.1:6379> hget ovimHashKey name
"ovim"
127.0.0.1:6379> hget ovimHashKey desc
"engineer"
127.0.0.1:6379> hgetall ovimHashKey
1) "name"
2) "ovim"
3) "desc"
4) "engineer"
127.0.0.1:6379>

实例中我们使用了 Redis hmset、hget、hgetall 命令，hmset 设置了一个 field=>value 对

每个 hash 可以存储 $2^32-1$ 键值对（40多亿即2的32次方-1）。

List（列表）​

详细介绍

Redis 列表是简单的字符串列表，按照插入顺序排序。你可以添加一个元素到列表的头部（左边 lpush ）或者尾部（右边 rpush ）。

命令

lpush key value [value ...]
lrange key start stop

实例

127.0.0.1:6379> lpush ovimListKey ovim
(integer) 1
127.0.0.1:6379> lpush ovimListKey is
(integer) 2
127.0.0.1:6379> lpush ovimListKey engineer
(integer) 3
127.0.0.1:6379> lrange ovimListKey 0 5
1) "engineer"
2) "is"
3) "ovim"
127.0.0.1:6379>

列表最多可存储 $2^32-1$ 元素 (4294967295, 每个列表可存储40多亿)。

Set（无序集合）​

详细介绍

Redis的Set是string类型的无序集合。

集合是通过哈希表实现的，所以添加，删除，查找的复杂度都是O(1)。

sadd 命令

添加一个 string 元素到 key 对应的 set 集合中，成功返回1，如果元素已经在集合中返回 0，如果 key 对应的 set 不存在则返回错误。

sadd key member
smembers key

实例

127.0.0.1:6379> sadd ovimCloud hello
(integer) 1     
127.0.0.1:6379> sadd ovimCloud i
(integer) 1     
127.0.0.1:6379> sadd ovimCloud am
(integer) 1     
127.0.0.1:6379> sadd ovimCloud ovim
(integer) 1     
127.0.0.1:6379> sadd ovimCloud ovim
(integer) 0     
127.0.0.1:6379> smembers ovimCloud
1) "ovim"       
2) "i"
3) "am"
4) "hello"      
127.0.0.1:6379>

注意：以上实例中 ovim 添加了两次，但根据集合内元素的唯一性，第二次插入的元素将被忽略。

集合中最大的成员数为 $2^32-1$ (4294967295, 每个集合可存储40多亿个成员)。

zset(sorted set：有序集合)​

详细介绍

Redis zset 和 set 一样也是string类型元素的集合,且不允许重复的成员。

不同的是每个元素都会关联一个double类型的分数。redis正是通过分数来为集合中的成员进行从小到大的排序。

zset的成员是唯一的,但分数(score)却可以重复。

命令

添加元素到集合，元素在集合中存在则更新对应score

zadd key score member
zrangebyscore key min max [WITHSCORES] [LIMIT offset count]

实例

127.0.0.1:6379> zadd ovimCloud 0 hello
(integer) 1     
127.0.0.1:6379> zadd ovimCloud 0 i
(integer) 1     
127.0.0.1:6379> zadd ovimCloud 0 am
(integer) 1     
127.0.0.1:6379> zadd ovimCloud 0 ovim
(integer) 1     
127.0.0.1:6379> zadd ovimCloud 0 ovim
(integer) 0     
127.0.0.1:6379> zrangebyscore ovimCloud 0 10
1) "am"
2) "hello"      
3) "i"
4) "ovim"       
127.0.0.1:6379>
